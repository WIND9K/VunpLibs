OnusLibs – Tổng hợp thông tin cập nhật (hybrid segment + DB + facade)

1. MỤC TIÊU CHUNG
- Cung cấp 1 facade duy nhất để lấy dữ liệu REST từ API Onus/Cyclos:
  - Hàm chính: onuslibs.unified.api.fetch_json()
- Ẩn toàn bộ phần phức tạp:
  - Token, header bảo mật (Keyring / ENV)
  - HTTP/2, timeout, retry/backoff (do HttpClient và httpx đảm nhiệm)
  - Rate-limit và song song (max_inflight, req_per_sec, parallel)
  - Phân trang (HeaderPager, page/pageSize, X-Page-Count, X-Total-Count)
  - Hybrid auto segment khi gặp lỗi 422 (giới hạn 10k records của API)
- Chuẩn hoá cấu hình bằng OnusSettings và .env
- Có module DB dùng chung (MySQL) lấy thông tin kết nối từ keyring.

2. KIẾN TRÚC MODULE CHÍNH
(Ở mức high-level, không chi tiết code)

2.1. config.settings.OnusSettings
- Đóng vai trò “runtime config”/“ENV loader” cho toàn bộ OnusLibs.
- Tự động:
  - Load .env (nếu có python-dotenv)
  - Đọc ENV ONUSLIBS_* và ép kiểu, validate.
- Các trường quan trọng:

  HTTP & runtime:
  - base_url           ← ONUSLIBS_BASE_URL (bắt buộc, http/https)
  - page_size          ← ONUSLIBS_PAGE_SIZE (mặc định 20000)
  - req_per_sec        ← ONUSLIBS_REQ_PER_SEC (mặc định 2.0)
  - max_inflight       ← ONUSLIBS_MAX_INFLIGHT (mặc định 4)
  - pager_parallel     ← ONUSLIBS_PARALLEL (song song trong 1 window)
  - timeout_s          ← ONUSLIBS_TIMEOUT_S (mặc định 60s)
  - http2              ← ONUSLIBS_HTTP2 (mặc định True)

  Hybrid segment (datePeriod):
  - date_segment_hours       (legacy, manual; mặc định 0 – không dùng)
  - max_window_days          ← ONUSLIBS_MAX_WINDOW_DAYS (mặc định 1)
  - auto_segment             ← ONUSLIBS_AUTO_SEGMENT (mặc định True)
  - max_rows_per_window      ← ONUSLIBS_MAX_ROWS_PER_WINDOW (mặc định 8000 – tạm thời chưa bật logic chia theo rows, nhưng đã có field)
  - max_segment_split_depth  ← ONUSLIBS_MAX_SEGMENT_SPLIT_DEPTH (mặc định 4)
  - segment_parallel         ← ONUSLIBS_SEGMENT_PARALLEL (mặc định True)
  - segment_max_workers      ← ONUSLIBS_SEGMENT_MAX_WORKERS (mặc định 0 → None → dùng max_inflight)

  Secrets & bảo mật:
  - secrets_backend   ← ONUSLIBS_SECRETS_BACKEND (keyring | env, default keyring)
  - keyring_service   ← ONUSLIBS_KEYRING_SERVICE (mặc định "OnusLibs")
  - keyring_item      ← ONUSLIBS_KEYRING_ITEM (mặc định "ACCESS_CLIENT_TOKEN")
  - fall_back_env     ← ONUSLIBS_FALLBACK_ENV (cho phép fallback đọc token từ ENV)
  - token_header      ← ONUSLIBS_TOKEN_HEADER (mặc định "Access-Client-Token")

  Một số field phụ:
  - log_level         ← ONUSLIBS_LOG_LEVEL (mặc định INFO)
  - user_agent        (tự sinh: OnusLibs/3 (Python X.Y))
  - proxy             ← ONUSLIBS_PROXY (tuỳ chọn)
  - verify_ssl        ← ONUSLIBS_VERIFY_SSL (mặc định True)

- Hàm to_dict() phục vụ debug/log.

2.2. http.HttpClient
- Dùng OnusSettings để cấu hình:
  - base_url, timeout, HTTP/2, rate limit, max_inflight…
- Cung cấp các phương thức:
  - get(path, params, headers)
  - post(path, json/data, headers), ...
- Tích hợp:
  - Timeout, retry (nếu config)
  - Rate-limit và giới hạn song song qua max_inflight, req_per_sec.

2.3. security.build_headers
- Dùng OnusSettings để:
  - Đọc token từ Keyring (theo keyring_service, keyring_item).
  - Nếu fall_back_env=True và không có keyring thì đọc từ ENV.
- Trả về dict headers:
  - { token_header: token_value, "User-Agent": user_agent, ... }

2.4. pagination.HeaderPager
- Xử lý phân trang chuẩn Cyclos:
  - Tham số page, pageSize.
  - Đọc header: X-Page-Count, X-Total-Count, X-Has-Next-Page…
- Hỗ trợ:
  - fetch_all(): generator trả về các batch list[dict].
  - Parallel paging trong 1 window:
    - Dùng ThreadPoolExecutor nếu parallel=True.

3. FACADE CHÍNH: unified.api.fetch_json

3.1. Chữ ký hàm
    fetch_json(
        endpoint: str,
        params: Optional[Dict[str, Any]] = None,
        *,
        fields: Optional[Sequence[str] | str] = None,
        page_size: Optional[int] = None,
        paginate: bool = True,
        order_by: Optional[str] = None,
        strict_fields: bool = False,
        unique_key: Optional[str] = None,
        settings: Optional[OnusSettings] = None,
        on_batch: Optional[Callable[[List[Dict[str, Any]]], None]] = None,
        client: Optional[HttpClient] = None,
        pager_func: Optional[Callable[[...], Iterable[List[Dict[str, Any]]]]] = None,
        extra_headers: Optional[Dict[str, str]] = None,
        parallel: Optional[bool] = None,
        workers: Optional[int] = None,
    ) -> List[Dict[str, Any]]

3.2. Tham số app gần như luôn truyền
- endpoint: chuỗi path tương đối (vd: "/api/transfers").
- params: dict filter, trong đó thường có:
  - "datePeriod": "start_iso,end_iso"
  - Các filter khác (transferTypes, user, amountRange...)
- fields: list hoặc chuỗi, sẽ tự join thành "a,b,c" và gắn vào params["fields"].
- unique_key: tên trường dùng để dedupe trên toàn bộ kết quả (vd "transactionNumber").

Các tham số còn lại thường giữ default, vì đã cấu hình trong OnusSettings/.env.

3.3. Hành vi khi KHÔNG có datePeriod hoặc paginate=False
- Nếu không có params["datePeriod"] hoặc paginate=False:
  - fetch_json chỉ gọi _fetch_single_window() đúng 1 lần:
    - Không hybrid segment.
    - Nếu paginate=True vẫn dùng HeaderPager nhưng không chia time window.
  - Kết quả: list[dict].

3.4. Hành vi khi CÓ datePeriod + paginate=True (hybrid segment)
- Đây là case chính cho các API transaction lớn.

Flow:
1) Parse datePeriod "start,end" → (start_dt, end_dt).

2) Tầng 1 – Window theo ngày (MAX_WINDOW_DAYS):
   - max_window_days = settings.max_window_days (default 1, có thể set bằng ONUSLIBS_MAX_WINDOW_DAYS).
   - Nếu max_window_days > 0:
     - Gọi _split_by_days(start_dt, end_dt, max_window_days)
     - Ví dụ: 01→11, MAX_WINDOW_DAYS=1 → 11 window, mỗi window = 1 ngày.
       MAX_WINDOW_DAYS=2 → ~6 window: [1–2], [3–4], …
   - Nếu max_window_days = 0:
     - Chỉ có 1 window: [start_dt, end_dt].

3) Tầng 2 – Segment theo giờ (date_segment_hours – legacy):
   - hours = settings.date_segment_hours (ONUSLIBS_DATE_SEGMENT_HOURS, default 0).
   - Với mỗi window (win_start, win_end):
     - Nếu hours > 0: _build_segments(win_start, win_end, hours) → chia đều theo giờ.
       (legacy/manual; có thể bỏ dùng dần).
     - Nếu hours = 0: window chính là 1 segment.
   - Kết quả: danh sách segments [(seg_start, seg_end), ...].

4) Tầng 3 – Auto-segment 422 theo thời gian (auto_segment):
   - Cấu hình:
     - auto_segment = settings.auto_segment (ONUSLIBS_AUTO_SEGMENT, default True)
     - max_split_depth = settings.max_segment_split_depth (default 4)
   - Với mỗi segment:
     - Nếu auto_segment=True và max_split_depth>0:
       - Gọi _run_with_split(seg_start, seg_end, depth=0)
       - _run_with_split:
         - Gọi _run_window(seg_start, seg_end).
         - Nếu thành công → trả list rows.
         - Nếu lỗi 422 ("Pagination error ... 422 ..."):
           - Nếu depth < max_split_depth:
             - Chia đôi khoảng thời gian [seg_start, seg_end] → [seg_start, mid], [mid, seg_end].
             - Gọi lại _run_with_split cho 2 nửa (đệ quy).
           - Nếu depth >= max_split_depth:
             - Log: "Auto-segment: reached max split depth=... nhưng API vẫn trả 422"
             - Raise exception.
     - Nếu auto_segment=False hoặc không phải 422 → raise luôn, không chia nhỏ.

5) Segment song song (segment_parallel):
   - Cấu hình:
     - segment_parallel = settings.segment_parallel (ONUSLIBS_SEGMENT_PARALLEL, default True).
     - segment_max_workers = settings.segment_max_workers (ONUSLIBS_SEGMENT_MAX_WORKERS, default 0 → None → dùng max_inflight).
   - Nếu segment_parallel=True và len(segments) > 1:
     - Dùng ThreadPoolExecutor để chạy _process_segment(seg) cho mỗi segment.
     - Số worker = segment_max_workers hoặc max_inflight.
   - Nếu False: chạy từng segment tuần tự.

6) Dedupe & merge kết quả:
   - Mỗi segment trả về list[dict] từ _run_window/_run_with_split.
   - _merge_batch:
     - Nếu unique_key được set (vd "transactionNumber"):
       - Giữ set "seen" ở mức toàn bộ fetch_json, dedupe cross-segment.
     - Gọi on_batch(batch) nếu có.
     - Append vào results.

Kết quả cuối cùng: list[dict] đã được:
- Gộp từ nhiều window/segment/time-split.
- Dedupe theo unique_key nếu có.
- Giữ nguyên thứ tự merge theo completion (song song) hoặc theo thứ tự segment (tuần tự).

4. HYBRID SEGMENT – TỔNG QUAN CẤU HÌNH

Các biến ENV chính:

- ONUSLIBS_MAX_WINDOW_DAYS
  - Default: 1.
  - 0 hoặc None: không chia theo ngày, toàn bộ datePeriod là 1 window.
  - >0: chia datePeriod thành nhiều window, mỗi window dài tối đa N ngày.

- ONUSLIBS_DATE_SEGMENT_HOURS
  - Default: 0.
  - 0: không chia theo giờ (mỗi window là 1 segment).
  - >0: legacy/manual – chia mỗi window thành các segment dài N giờ.

- ONUSLIBS_AUTO_SEGMENT
  - Default: True.
  - True: bật auto-split khi gặp 422 trong 1 segment (chia đôi theo thời gian, tối đa depth).
  - False: tắt auto-split – nếu 422 xảy ra sẽ raise lỗi thẳng.

- ONUSLIBS_MAX_SEGMENT_SPLIT_DEPTH
  - Default: 4.
  - Số tầng tối đa cho _run_with_split (đệ quy).
  - depth=0 → 1 window; depth=4 → tối đa 16 cửa sổ con cho cùng khoảng thời gian.

- ONUSLIBS_SEGMENT_PARALLEL
  - Default: True.
  - True: chạy nhiều segment (window/time-split) song song bằng ThreadPoolExecutor.
  - False: chạy từng segment tuần tự.

- ONUSLIBS_SEGMENT_MAX_WORKERS
  - Default: 0 → None (dùng max_inflight).
  - >0: giới hạn số thread xử lý segment song song.

- ONUSLIBS_PARALLEL
  - Điều khiển parallel bên trong 1 window khi phân trang (HeaderPager).

6. MODULE DB CỦA ONUSLIBS

Mục tiêu:
- 1 module DB đơn giản, dùng chung cho các app OnusReport / script.
- Tất cả thông tin kết nối lấy từ keyring (hoặc ENV fallback).

6.1. Cấu hình DbSettings (trong onuslibs.db.core)
- Các field điển hình:
  - host, port, user, password, db_name, ssl_ca…
- Hàm from_secure() đọc từ keyring với service "OnusLibs" và các key:
  - DB_HOST, DB_USER, DB_PASSWORD, DB_NAME, DB_PORT, DB_SSL_CA.

Ví dụ set keyring trên Windows (PowerShell):

  $svc="OnusLibs"
  python -c "import keyring; s='$svc'; keyring.set_password(s,'DB_HOST','localhost')"
  python -c "import keyring; s='$svc'; keyring.set_password(s,'DB_USER','onusreport')"
  python -c "import keyring; s='$svc'; keyring.set_password(s,'DB_PASSWORD','xxx')"
  python -c "import keyring; s='$svc'; keyring.set_password(s,'DB_NAME','onusreport')"
  python -c "import keyring; s='$svc'; keyring.set_password(s,'DB_PORT','3306')"
  python -c "import keyring; s='$svc'; keyring.set_password(s,'DB_SSL_CA','')"

6.2. Các hàm DB chính (onuslibs.db)
- connect(): trả về connection MySQL (mysql.connector hoặc tương tự).
- healthcheck(): thử kết nối DB, trả về True/False hoặc raise.
- query(sql, params=None): trả list[dict] (hoặc list tuple) cho SELECT.
- execute(sql, params=None): chạy INSERT/UPDATE/DELETE đơn lẻ.
- bulk_insert(table, rows): insert nhiều dòng 1 lần.
- transactional decorator (nếu có): giúp wrap block code trong transaction.

9. TÓM TẮT NGẮN CHO APP

Khi build 1 app mới dùng OnusLibs, pattern chuẩn:

1) Chuẩn bị .env theo khuyến nghị (BASE_URL, PAGE_SIZE, REQ_PER_SEC, MAX_INFLIGHT, PARALLEL, SEGMENT_PARALLEL, AUTO_SEGMENT…).
2) Thiết lập token + DB credentials trong keyring (service=OnusLibs).
3) Trong app:
   - from onuslibs.unified.api import fetch_json
   - from onuslibs.utils.date_utils import build_date_period

   date_period = build_date_period(start_date, end_date)
   params = {
       "datePeriod": date_period,
       "...": "...",
   }
   rows = fetch_json(
       endpoint="/api/xxx",
       params=params,
       fields=[...],
       unique_key="transactionNumber",  # nếu có
   )

4) Nếu cần DB:
   - from onuslibs.db import connect, query, execute, bulk_insert
   - conn = connect()  # dùng DbSettings.from_secure()

Như vậy, OnusLibs giữ đúng mục tiêu: 1 facade REST-first (fetch_json) + 1 module DB, còn lại mọi logic auto segment, pagination, rate-limit, bảo mật đều chạy ngầm.
